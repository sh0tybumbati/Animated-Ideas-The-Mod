# GarrettTheCarrotMod Instructions

## Architectural Principles
- **Modularity:** Every feature should be its own class/package to allow easy toggling via configuration.
- **Fidelity to Animation:** Visuals and behaviors should match GarrettTheCarrot's animations as closely as possible.

## Tech Stack (Planned)
- **Language:** Java
- **Target:** Minecraft (Latest Stable)
- **Modding API:** Fabric (Preferred for lightweight/modern features)

## Workflows
- Always update `PLAN.md` when a new idea is selected for implementation.
- Features should be implemented in small, testable increments.
