# Project Plan - GarrettTheCarrotMod

## Feature Backlog

Based on GarrettTheCarrot's series, here are the initial ideas categorized for implementation:

### 🌟 Quality of Life / "Good" Ideas
- **Placeable Gunpowder:** Create fuses on the ground that can be lit with flint and steel.
- **Repairing Anvils:** Use iron ingots on a damaged anvil to restore its durability.
- **Cooperative Mining:** Blocks break faster when multiple players mine the same block simultaneously.
- **Milk Cauldrons:** Pour milk into cauldrons; potentially leads to cheese production.
- **Trident Hitboxes:** Make tridents solid when stuck in blocks, allowing them to be used as platforms.
- **Jungle Villages:** Unique architectural style for villages in jungle biomes.
- **Parrots on Armor Stands:** Allow parrots to be placed on armor stands.
- **Slime Ball Throwing:** Throwing slime balls slows down entities or makes them "sticky".
- **Green Apples:** A new food item variant.
- **Full Turtle Armor:** Complete the set with chestplate, leggings, and boots made from scutes.
- **Mixed Slabs:** Ability to place two different types of slabs in the same block space.

### 🌀 "Worst" / Chaotic Ideas
- **Windy Redstone:** Redstone dust has a chance to blow away or move during "windy" weather (to be defined).
- **Boomerang Arrows:** Arrows that return to the shooter (and might hit them).
- **Layered Cakes:** Stack cakes on top of each other.
- **Creeper Mercenaries:** "Tame" creepers with emeralds to target specific locations or players.
- **Air Sheep:** Rare occurrence where air blocks "transform" into sheep.

## Technical Roadmap
1.  **Phase 1: Research & Selection** (Current)
    - Finalize the list of features for the MVP (Minimum Viable Product).
    - Choose the modding API (Fabric vs Forge/NeoForge).
2.  **Phase 2: Environment Setup**
    - Initialize the Gradle project.
    - Set up assets (textures, models).
3.  **Phase 3: Core Implementation**
    - Implement the first set of "Good Ideas".
4.  **Phase 4: Chaos Implementation**
    - Add the "Worst Ideas" as optional/config-driven features.
